### 2.2 본질주의(Essentialism) UI 설계 — 핵심 골격 확정

Aurora v1의 한계를 진단하고 본질주의 원칙으로 재설계:

**v1의 실패:** 데이터베이스 브라우저였음. 그래프를 전면 배치하고 사용자에게 "알아서 탐색하세요"라고 함. 자기증명시스템이 아니었음.

**v2의 3원칙:**
1. **시스템이 먼저 말한다** — 첫 화면에서 "7-layer 범죄 은폐 구조"를 3초 안에 선언
2. **모순이 기본이다** — 기본 뷰가 모순 쌍(contradiction pair), 그래프가 아님
3. **증거와의 거리는 0** — Record No. 스캔본이 항상 우측에 존재

**10개 불변 골격 요소 확정:**
1. Landing: 7-Layer 탑 + 진행률 + 통계 + 단일 입력
2. 3-Panel: Layer Navigator / 모순 쌍 / 증거 맥락
3. 그래프는 on-demand 모달
4. 모순 쌍이 기본 표시 단위
5. 증거 거리 0 (Record No. 항상 표시)
6. 명시적 모드 토글 없음 — 맥락적 전환
7. 3축 진실 배지
8. 역할-앵커(역할-N) 그래프 노드 라벨
9. Proof Level 시각화 (3-level Relationship filter)
10. 정적 JSON 그래프 (Neo4j 독립)

James가 이 골격의 **불변성을 명시적으로 요청** — 향후 기능 추가는 골격 위에 층을 쌓는 것만 허용.

### 2.3 시각화 도구 비교 및 Proof Level 설계

5개 시각화 도구를 비교 평가하고 PPT 시안 생성:

| Variant | 도구 | 판정 |
|---|---|---|
| A | Cytoscape.js (v1 실제 코드) | ALREADY BUILT |
| B | Obsidian Constellation (PIXI.js) | INSPIRATION |
| C | G6/AntV | RECOMMENDED (프로덕션) |
| D | 3D Force Graph (Three.js) | WOW FACTOR (프레젠테이션) |
| E | Sigma.js/Gephi | SCALE CHAMPION |

**최종 결정:** Cytoscape.js 단일 도구로 3개 Proof Level 모두 커버.

> "Cytoscape.js를 유지하되, '전부 보여주기'에서 '증명 단위별로 보여주기'로 전환하라. 도구를 바꾸는 것이 아니라, 같은 도구로 보여주는 관계를 제한하는 것이 본질주의다."

**Proof Level 설계:**
- Level 1 (범죄 구조도): ORDERED_BY, EXECUTED_BY, VICTIMIZED, CRIME_CHAIN → COSE
- Level 2 (증거 연결도): SUPPORTED_BY, VIOLATED, MANIPULATED_WITH → Breadthfirst
- Level 3 (교차 층위 조감): CRIME_CHAIN, PART_OF_LAYER, CONCEALED_BY → Concentric

### 2.4 Neo4j 스키마 진화 결정

Atom 완성 시 Neo4j 스키마 변경 필요 항목 도출:

**추가 (5건):**
1. `OPPOSES` 관계 (FR↔FR) — **Checkpoint 2 핵심**, 모순 쌍 표현
2. `roleAnchors` 속성 (Person) — 역할-앵커 `(역할-N)`
3. `CITES_RECORD` 관계 (FR→Evidence) — Record No. 직접 연결
4. `status` 속성 (FR) — DRAFT/VERIFIED/DISPUTED
5. `wikiPath`/`wikiSlug` 속성 (FR) — UI→wiki 연결

**보완 (3건):** Evidence `recordNo`, `SUPERSEDES` 체인 보강, `TARGETS` 검증
**삭제:** 없음 (Community 구조는 자연 소멸 대기)

### 2.5 정적 그래프 아키텍처 — Neo4j 독립 (James 발의)

James가 "Neo4j에서 가져오지 않고 atom과 매핑된 nodes/relationships를 별도로 구축"을 제안. Claude가 동의하며 구체화:

**기존:** atoms → Neo4j → API 쿼리 → UI (런타임 의존)
**변경:** atoms → graph.json (빌드 시 정적) → UI 직접 로드 (서버 불필요, $0)

Neo4j는 critical path에서 빠짐. 고급 분석(multi-hop traversal, Text2Cypher, FalsificationChecker)에서만 선택적 사용.

**비용 영향:**
- 기존: Docker(Neo4j+API+UI) + $15~50/월 웹 배포
- 변경: UI만 (정적 호스팅 $0), Neo4j는 분석 세션에서만 로컬 Docker
