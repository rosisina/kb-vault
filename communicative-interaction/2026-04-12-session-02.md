### 2.2 "다크 기본 + Claude.ai 구조 차용" 확정 (James 승인)

James가 Claude.ai 동일 스타일과 기존 은은한 다크를 비교 요청. Claude가 3개 방안 비교 후 1순위 추천:

> **Claude.ai의 행동(구조, 인터랙션, 여백)은 가져오되, 외양(색상, 톤)은 Aurora 고유를 유지한다.**

| Claude.ai에서 차용 | Aurora 고유 유지 |
|---|---|
| 입력창: 라운드 24px, 중앙 하단, 미세 테두리 | 다크 배경 (#0f1117) |
| 여백: 넓은 마진, 콘텐츠 최대 너비 제한 | 7-Layer 은은한 색상 |
| 인터랙션: 호버 시 rgba 변화만 | 모순 쌍 카드, 3축 배지 |
| 메뉴: 화면에 보이는 버튼 ≤ 3 | Proof Level 시각화 |

**이유:** Claude.ai는 "대화 도구" — 따뜻함이 적합. Aurora v2는 "증명 시스템" — 전문성과 권위 필요. 동일하면 브랜드 혼동.

구체적 구현 세부사항 구현 가이드에 반영:
- `.interactive-element` 호버: `rgba(255,255,255,0.04)` 미세 변화만
- `.card` 테두리: `rgba(255,255,255,0.04)`, 그림자 없음
- 스크롤바: 기본 숨김, 호버 시 은은하게
- 토스트/알림: 없음 (Claude.ai 동일)
- CSS 변수 기반 — 향후 라이트 모드 추가 가능 (골격 불변)

### 2.3 다국어 지원 — 3-tier 언어 구조 (James 승인)

Aurora v1의 인라인 삼항 방식(`lang() === 'en' ? 'X' : 'Y'`, 23개 파일 분산)을 JSON 번역 파일 방식으로 개선:

| Tier | 대상 | 전환 방식 |
|---|---|---|
| Tier 1: UI 라벨 | 메뉴, 버튼 | KR/EN 토글 즉시 |
| Tier 2: 도메인 용어 | Layer 이름, verdict | KR/EN 토글 즉시 |
| Tier 3: 콘텐츠 | atom 본문, verbatim | **전환 안 함 — 원본 보존** |

구현: `assets/i18n/kr.json` + `en.json`, `LanguageService.t(key)` 함수, 기본 언어 한국어.
향후 다국어(일본어, 중국어 등) 확장: JSON 파일 하나 추가만.

### 2.4 개방형 웹 전환 승인 (James 발의 + 승인)

James가 "전 세계 사람들이 사용 가능하도록 개방형 개발 필요"라고 판단. Claude가 동의하며 단계적 개방 로드맵 제시:

> **"증명 시스템은 공개되어야 증명으로 기능한다."**

**4단계 개방:**
- Stage 0 (현재): James 단독
- Stage 1 (Checkpoint 2): 읽기 전용 공개, 정적 사이트, $0
- Stage 2: 클라이언트 사이드 BM25 검색 추가, $0
- Stage 3: LLM 질의 (선택적, 비용 발생)

**핵심 근거:**
1. 정적 JSON 아키텍처로 추가 비용 $0
2. 서버 없으므로 보안 표면 최소
3. 3-tier 언어 구조로 다국어 이미 준비됨
4. 모든 추가 사항이 골격 위 추가

**가명화 3중 방어:** 빌드 시 pseudonym 대조(실명→빌드실패) + 런타임 guard + CSP 헤더.

구현 가이드에 Phase 4로 신설 (작업 4a~4h).
